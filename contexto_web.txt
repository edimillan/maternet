1. CARACTERÍSTICAS ACTUALES DEL SISTEMA (¿Cómo es la Web?)
----------------------------------------------------------------------
- Tipo de Arquitectura: Single Page Application (SPA) dinámica. La cabecera (Header) y el pie de página (Footer) están fijos en "index.html" y el contenido cambia dentro de un contenedor central mediante JavaScript (Fetch/AJAX) con cache-buster dinámico (`?v=timestamp`) para evitar cacheo de archivos en cambios del CMS.
- Enrutamiento por Anclas Inteligente: El motor SPA (`loadPage`) soporta anclajes (`#id-seccion`). Si el usuario hace clic en un menú de la barra de navegación (como "Niveles"), la web hace un desplazamiento suave hacia el bloque correspondiente.
- Estilo y Diseño Premium:
  * Paleta oficial: Verde principal (`#005931`), Oro (`#EBB300`), Fondo Neutro (`#f8f9fa`) y Texto Oscuro (`#1A252C`).
  * Fondos "Aurora" Animados: Blobs difuminados flotantes (verde y oro al 5% con `blur-3xl`) que flotan dinámicamente mediante animaciones CSS optimizadas en cada sección.
  * Paralaje Interactivo (Anime.js): Los blobs del fondo reaccionan al movimiento físico del mouse desplazándose suavemente en 3D.
  * Transiciones de Elevación y Sombra Nativas: Las tarjetas informativas principales del Home y niveles reaccionan con un sutil e interactivo levantamiento y sombra al pasar el cursor manejados por hardware mediante CSS Transitions, eliminando bloqueos de JavaScript.
  * Micro-interacción de Elementos Decorativos: Animación CSS `@keyframes float-soft` que añade un vaivén flotante constante a badges e íconos decorativos.
- Coherencia Institucional de Género: Auditoría completa de textos. El lenguaje se adaptó al perfil del colegio (mujeres en Primaria y Secundaria: "alumnas", "hijas", "líderes analíticas y ciudadanas"). En Nivel Inicial (3 y 4 años) se especifica explícitamente la coeducación (niños y niñas), mientras que el aula de 5 años es exclusiva para niñas.

2. MAPA DE ARCHIVOS Y COMPONENTES
----------------------------------------------------------------------
Todos los archivos secundarios son parciales limpios que se inyectan en `<main>` de forma dinámica y reactiva:
- index.html -> Contenedor principal, cabecera fija, barra superior, menú móvil tipo acordeón interactivo, pie de página maestro, motor SPA con cache-buster, y metaetiquetas SEO Open Graph para compartir enlaces.
- inicio.html -> Página de bienvenida en bloques anclados:
  * Banner Mosaico (Carga dinámica de 6 a 8 imágenes WebP distribuidas asimétricamente)
  * Proyecto Institucional (`#institucional-block` -> Enlaces a Presentación, Misión, IB e Historia)
  * Nuestros Niveles Educativos (`#niveles-block` -> Inicial, Primaria, Secundaria en proporción aspect-[4/3])
  * Convenios y Alianzas (`#convenios-block` -> Académicos, Alianza Francesa, Cambridge)
  * Servicios de la Comunidad (`#servicios-block` -> Psicopedagógico, Escuela para padres, Comedor)
  * Comunidad y Admisión (`#comunidad-block` -> Noticias y actividades reales conectadas a JSON, y admisión rápida con Netlify Forms).
- niveles/inicial.html -> Detalle dinámico del Nivel Inicial conectado a `data/nivel_inicial.json`.
- niveles/primaria.html -> Detalle dinámico de Primaria conectado a `data/nivel_primaria.json` (incluye sección especial de inglés).
- niveles/secundaria.html -> Detalle dinámico de Secundaria conectado a `data/nivel_secundaria.json` (incluye tarjeta destacada con logotipo oficial de Bachillerato Internacional).
- admision.html -> Proceso de postulación completo de forma dinámica mediante `data/admision.json`.
- contacto.html -> Datos de localización, canales, mapa del campus y formulario dinámico enlazado a Netlify Forms.
- comunidad/detalle.html -> Plantilla de detalle de noticias dinámico con lightbox de fotos y hoja de estilos de impresión oficial optimizada para boletines en PDF/Papel.
- 404.html -> Página de error personalizada con el branding del colegio y redireccionamiento automático.

3. DATA JSON E INTEGRACIÓN DE DECAP CMS
----------------------------------------------------------------------
Los contenidos de la web se gestionan a través de archivos de datos que lee JavaScript:
- data/comunidad.json -> Colección de noticias y próximas actividades del colegio.
- data/inicio.json -> Configuración de imágenes del mosaico de inicio y fotos/textos de los niveles de la portada.
- data/admision.json -> Contenido oficial del proceso de admisión.
- data/nivel_inicial.json / nivel_primaria.json / nivel_secundaria.json -> Textos, mallas curriculares e imágenes de cada nivel educativo.
- admin/config.yml -> Configuración del CMS Decap usando el backend directo de GitHub (`name: github`) enlazado a los endpoints de la API de Cloudflare.
- admin/index.html -> Panel administrativo del CMS con estilos e Inter/Tailwind inyectados para previsualización.
- functions/api/auth.js / callback.js -> Endpoints serverless creados para Cloudflare Pages para procesar la autorización del login mediante OAuth de GitHub de manera segura sin Netlify.
